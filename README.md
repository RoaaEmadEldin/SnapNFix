# SnapNFix System

SnapNFix is a smart urban issue reporting system designed to improve civic engagement and infrastructure maintenance by connecting citizens with local authorities through a unified digital platform.

The system consists of two integrated components:

* **Citizen Mobile Application** for reporting and tracking urban issues
* **Administrative Dashboard** for managing incidents, users, and analytics

---

## Repositories

- Mobile Application: [SnapNFix Mobile](https://github.com/RoaaEmadEldin/SnapNFix-Mobile)
- Admin Dashboard: [SnapNFix Dashboard](https://github.com/RoaaEmadEldin/SnapNFix-Dashboard)

---

## System Overview

SnapNFix enables citizens to report issues such as:

* Potholes
* Broken manholes
* Garbage accumulation
* Street infrastructure damage

Reports are submitted with images, location data, and issue details, allowing administrators to review, track, and resolve incidents efficiently.

---

## Components

### Mobile Application

A Flutter-based application for citizens to:

* Capture and report issues with location tagging
* Track issue status in real-time
* View nearby incidents on an interactive map
* Subscribe to area updates
* Receive push notifications
* Use the app offline with synchronization support

**Tech Stack:**

* Flutter
* Dart
* BLoC
* Hydrated BLoC
* Firebase
* Dependency Injection
* Retrofit
* Freezed

Repository: [SnapNFix Mobile](https://github.com/RoaaEmadEldin/SnapNFix-Mobile)

---

### Admin Dashboard

A React + TypeScript dashboard for administrators to:

* Monitor incident metrics and trends
* Manage reported issues
* Manage admin and citizen accounts
* View analytics and category distribution
* Support Arabic and English interfaces
* Use role-based access control

**Tech Stack:**

* React
* TypeScript
* Redux Toolkit
* Tailwind CSS
* Firebase
* Axios
* React Router

Repository: [SnapNFix Dashboard](https://github.com/RoaaEmadEldin/SnapNFix-Dashboard)

## Architecture

SnapNFix follows a multi-platform system architecture:

* Mobile app for citizen interaction
* Dashboard for administrative operations
* Shared backend services
* Real-time synchronization
* Cloud messaging and notifications
